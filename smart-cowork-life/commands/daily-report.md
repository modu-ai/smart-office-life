사용자가 제공한 업무 내용을 바탕으로 한국 직장 문화에 맞는 일일업무보고서를 작성한다.

## 보고서 구조

1. **금일 업무 실적** — 완료한 업무를 중요도 순으로 정리
2. **진행 중 업무** — 현재 진행 상황과 예상 완료일
3. **익일 업무 계획** — 내일 할 일 우선순위 정리
4. **특이사항/건의사항** — 이슈, 리스크, 요청사항

## 작성 규칙

- 경어체 사용 (습니다/했습니다)
- 각 업무 항목은 "~완료", "~진행중(00%)", "~예정"으로 상태 표기
- 수치화 가능한 성과는 반드시 숫자 포함
- 간결하고 명확한 문장 (한 문장 30자 이내 권장)

사용자 입력: $ARGUMENTS

입력이 간단한 메모 형태라도 위 구조에 맞게 전문적인 보고서로 변환한다.
출력 형식은 마크다운으로 작성하되, 사용자가 워드나 한글 파일을 원하면 해당 형식으로 저장한다.
