# 프롬프트 엔지니어링 마스터 스킬

> AI를 "총명하지만 기억력이 없는 신입사원"으로 대하라 — Anthropic 프롬프트 가이드

## 1. 프롬프트 설계 5원칙

| 원칙 | 설명 | 예시 |
|------|------|------|
| **명확성(Clarity)** | 모호한 표현 제거, 구체적 지시 | ❌ "좋은 글 써줘" → ✅ "B2B SaaS 마케터 대상 블로그 글 1500자, 문제-해결-CTA 구조로" |
| **맥락(Context)** | 배경, 대상, 목적 제공 | "나는 5년차 마케터이고, 이 보고서는 C레벨 경영진에게 보고용" |
| **구조(Structure)** | 출력 형식을 지정 | "표 형식으로 / JSON으로 / 불릿포인트 5개로" |
| **제약(Constraints)** | 범위·분량·톤 제한 | "500자 이내, 존댓말, 전문용어 최소화" |
| **예시(Examples)** | 원하는 결과의 샘플 제공 | "다음과 같은 형식으로: [예시 삽입]" |

## 2. 핵심 프롬프트 기법 카탈로그

### 2.1 제로샷(Zero-shot)
예시 없이 명령만으로 결과를 도출합니다.

```
다음 텍스트의 감정을 분석하세요: "오늘 회의 정말 힘들었어"
→ 출력: 부정(피로/스트레스)
```

**적합한 상황**: 단순 분류, 번역, 요약 등 명확한 작업

### 2.2 퓨샷(Few-shot)
1~5개 예시를 제공하여 패턴을 학습시킵니다.

```
다음 형식으로 고객 문의를 분류하세요:

[예시1] "배송이 안 왔어요" → 카테고리: 배송, 긴급도: 높음
[예시2] "색상 변경 가능한가요?" → 카테고리: 상품문의, 긴급도: 낮음

[실제 문의] "결제했는데 취소하고 싶어요" →
```

**적합한 상황**: 특정 포맷 준수, 분류 기준이 복잡한 경우

### 2.3 체인오브소트(Chain-of-Thought, CoT)
단계별 사고 과정을 유도하여 정확도를 높입니다.

```
다음 문제를 단계별로 풀어주세요:

매출 1억 원, 매입 6천만 원, 인건비 2천만 원, 임차료 500만 원일 때
영업이익률은?

1단계: 총비용 계산
2단계: 영업이익 계산
3단계: 영업이익률(%) 계산
```

**적합한 상황**: 수학/재무 계산, 논리 추론, 복잡한 분석

### 2.4 역할지정(Role Prompting)
AI에게 전문가 역할을 부여합니다.

```
당신은 10년 경력의 한국 세무사입니다.
1인기업 대표(연매출 4800만 원, 간이과세자)의 종합소득세 절세 전략을
실무적 관점에서 5가지 추천해주세요.
```

**역할 템플릿 모음**:

| 업무 | 역할 | 추가 맥락 |
|------|------|-----------|
| 세무 | 10년차 세무사 | 업종, 매출규모, 과세유형 |
| 법률 | 기업전문 변호사 | 계약 유형, 분쟁 상황 |
| 마케팅 | 퍼포먼스 마케터 | 예산, 채널, KPI |
| 인사 | HR 매니저 | 회사 규모, 채용 포지션 |
| 기획 | 전략기획 팀장 | 산업, 경쟁사, 목표 |
| 글쓰기 | 카피라이터 | 타겟, 톤앤매너, 매체 |

### 2.5 트리오브소트(Tree of Thoughts, ToT)
CoT의 확장. 여러 사고 경로를 탐색한 뒤 최적을 선택합니다.

```
우리 쇼핑몰의 매출을 올리기 위한 전략을 3가지 경로로 탐색하세요:

경로 A: 신규 고객 유치 관점
경로 B: 기존 고객 재구매율 관점
경로 C: 객단가 상승 관점

각 경로별로 장단점과 예상 ROI를 분석한 뒤, 최적 경로를 추천하세요.
```

### 2.6 메타 프롬프트
AI가 프롬프트 자체를 개선하도록 요청합니다.

```
나는 지금부터 정부지원사업 사업계획서를 쓰려고 합니다.
이 목적에 가장 효과적인 프롬프트를 설계해주세요.
필요한 정보가 있으면 질문해주세요.
```

### 2.7 환각 방지(Hallucination Prevention)

```
모르는 내용은 "확인이 필요합니다"라고 답하세요.
추측하지 말고, 출처를 명시하세요.
수치 데이터는 반드시 근거를 함께 제시하세요.
```

## 3. 업무별 프롬프트 레시피

### 3.1 보고서 작성
```
역할: 전략기획팀 과장
배경: [프로젝트명]의 중간보고를 위한 문서
대상: 본부장(비기술직)
구조: 1) 핵심 요약(3줄) 2) 진행현황 3) 이슈사항 4) 향후 계획
분량: A4 2페이지 이내
톤: 격식체, 데이터 기반
```

### 3.2 이메일 작성
```
역할: 영업팀 담당자
상황: 견적서를 보내면서 미팅 일정을 잡으려는 이메일
격식: 높음 (거래처 부장님 대상)
포함사항: 인사 → 견적 안내 → 미팅 제안(2~3개 시간대) → 마무리
분량: 200자 내외
```

### 3.3 데이터 분석
```
다음 매출 데이터를 분석해주세요: [데이터 붙여넣기]

분석 순서:
1단계: 데이터 품질 확인(결측치, 이상치)
2단계: 기술통계(평균, 중앙값, 표준편차)
3단계: 트렌드 분석(전월 대비, 전년 동기 대비)
4단계: 인사이트 도출(원인 분석)
5단계: 액션 제안(3가지)
```

## 4. 프롬프트 체이닝(Chaining)

복잡한 작업을 여러 프롬프트로 분해합니다.

```
[프롬프트 1] 시장조사 → 결과를 [변수1]에 저장
[프롬프트 2] [변수1] 기반으로 경쟁사 분석
[프롬프트 3] 경쟁사 분석 결과를 바탕으로 차별화 전략 수립
[프롬프트 4] 전략을 PPT 슬라이드 구조로 정리
```

## 5. 프롬프트 라이브러리 구축 가이드

### 5.1 템플릿 표준 구조
```markdown
## [프롬프트 이름]
- 목적:
- 역할:
- 입력 변수: {변수1}, {변수2}
- 출력 형식:
- 제약 조건:
- 프롬프트 본문:
  ...
- 예상 결과:
  ...
```

### 5.2 팀 공유 프로세스
1. 개인이 유용한 프롬프트 발견 → 사내 Wiki/노션에 등록
2. 주간 단위로 베스트 프롬프트 선정
3. 부서별 프롬프트 라이브러리 구축
4. 분기별 효과 측정 및 개선

## 6. 플랫폼별 특화 팁

| 플랫폼 | 핵심 팁 |
|--------|---------|
| **Claude** | XML 태그 활용(`<context>`, `<instructions>`), 긴 문서 분석에 강점, "차근차근" 키워드 효과적 |
| **ChatGPT** | Custom Instructions 활용, GPTs로 반복 프롬프트 자동화, 플러그인 연동 |
| **Gemini** | 멀티모달(이미지+텍스트) 분석, Google Workspace 연동 |
| **Midjourney/DALL-E** | 스타일 키워드(photorealistic, watercolor), 구도/조명 명시, 네거티브 프롬프트 |

## 7. 자주 하는 실수 & 해결법

| 실수 | 해결법 |
|------|--------|
| 너무 짧은 프롬프트 | 맥락+역할+구조+제약 추가 |
| 한 프롬프트에 여러 작업 | 작업 분리 후 체이닝 |
| 결과가 너무 일반적 | 구체적 예시 추가(퓨샷) |
| AI가 거짓 정보 생성 | 환각 방지 조건 추가 + 출처 요구 |
| 매번 같은 프롬프트 반복 | 프롬프트 라이브러리 구축 |
| 한국어 결과 품질 저하 | 핵심 키워드는 영어로, 나머지는 한국어로 |
