# 한국형 비즈니스 문서 생성기

## Overview

한국 직장인이 가장 많이 작성하는 비즈니스 문서를 자동 생성합니다.
모든 문서는 한국 기업 문화에 맞는 격식체와 양식을 따릅니다.

## 지원 문서 유형

| 문서 유형 | 용도 | 필수 항목 |
|-----------|------|-----------|
| 주간보고서 | 주간 업무 성과 보고 | 금주 실적, 차주 계획, 이슈사항 |
| 업무일지 | 일일 업무 기록 | 날짜, 업무내용, 진행상태, 비고 |
| 기안서 | 업무 추진 승인 요청 | 제목, 기안자, 기안일, 내용, 결재선 |
| 품의서 | 예산/구매 승인 요청 | 품의 내용, 금액, 사유, 결재선 |
| 지출결의서 | 비용 지출 승인 | 지출항목, 금액, 증빙, 결재선 |
| 출장보고서 | 출장 결과 보고 | 출장지, 기간, 목적, 결과, 경비 |
| 회의록 | 회의 내용 기록 | 일시, 참석자, 안건, 결정사항, 후속조치 |

## 작성 원칙

### 1. 문체 규칙
- **격식체 사용**: "~합니다", "~입니다" 체계 (구어체 금지)
- **간결한 문장**: 한 문장 40자 이내 권장
- **수치 중심**: 추상적 표현 대신 구체적 수치 사용
  - ❌ "매출이 많이 증가했습니다"
  - ✅ "매출이 전월 대비 15.3% 증가했습니다 (1.2억 → 1.38억)"
- **기-승-전-결 구조**: 배경→현황→분석→제안 순서

### 2. 결재선 포맷
```
┌──────┬──────┬──────┬──────┐
│ 담당  │ 팀장  │ 부장  │ 이사  │
├──────┼──────┼──────┼──────┤
│      │      │      │      │
│ (인)  │ (인)  │ (인)  │ (인)  │
└──────┴──────┴──────┴──────┘
```

### 3. 날짜/번호 표기
- 날짜: 2026. 02. 14. (마침표 구분)
- 문서번호: [부서코드]-[년도]-[일련번호] (예: HR-2026-0042)

## 문서별 양식 가이드

### 주간보고서

```
[문서 제목] 제N주차 주간업무보고 (MM.DD ~ MM.DD)

[작성자 정보]
부서: ___  직급: ___  성명: ___  작성일: ____. __. __.

[1. 금주 실적]
| No | 업무 항목 | 계획 | 실적 | 달성률 | 비고 |
|----|----------|------|------|--------|------|
| 1  |          |      |      |   %    |      |

[2. 차주 계획]
| No | 업무 항목 | 목표 | 일정 | 담당자 | 비고 |
|----|----------|------|------|--------|------|
| 1  |          |      |      |        |      |

[3. 이슈 및 건의사항]
- 이슈 1: (내용) → 조치계획: (대응 방안)
- 건의 1: (내용) → 기대효과: (효과)
```

```
[결재]
┌──────┬──────┬──────┐
│ 기안  │ 검토  │ 승인  │
├──────┼──────┼──────┤
│      │      │      │
└──────┴──────┴──────┘

제목: ________________________________________________

1. 목적
   (업무 추진의 필요성과 배경을 기술)

2. 현황
   (현재 상황 및 문제점을 구체적으로 기술)

3. 추진 내용
   가. (세부 추진 항목 1)
   나. (세부 추진 항목 2)

4. 소요 예산
   | 항목 | 금액(원) | 비고 |
   |------|---------|------|
   | 합계 |         |      |

5. 기대 효과
   (추진 시 예상되는 효과를 정량적/정성적으로 기술)

6. 향후 일정
   | 단계 | 내용 | 기간 | 담당 |
   |------|------|------|------|

붙임. 관련 자료 _부. 끝.
```

## 출력 형식

사용자가 원하는 형식에 따라 출력합니다:
- `.docx`: docx 스킬을 활용하여 Word 문서 생성 (기본값)
- `.hwpx`: hwpx 스킬을 활용하여 한글 문서 생성
- `.pdf`: pdf 스킬을 활용하여 PDF 생성
- `.md`: Markdown 텍스트로 출력

## 워크플로우

```
사용자 요청 → 문서 유형 파악 → 필요 정보 확인
→ 양식에 맞춰 문서 생성 → 출력 형식 변환 → 전달
```

### 정보 수집 가이드
문서 생성 시 최소한 다음 정보를 확인합니다:
1. **문서 유형** (보고서/기안서/품의서 등)
2. **작성자 정보** (이름, 부서, 직급)
3. **핵심 내용** (무엇을, 왜, 어떻게)
4. **출력 형식** (docx/hwpx/pdf/md)

사용자가 정보를 충분히 제공한 경우 바로 생성하고, 부족한 경우 필수 항목만 간단히 확인합니다.
