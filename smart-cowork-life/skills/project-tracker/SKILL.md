# 프로젝트 관리 트래커 스킬

## 1. WBS(Work Breakdown Structure) 작성

### 1.1 WBS 표준 구조

```
프로젝트 (Level 0)
├── 1. 주요 단계 (Level 1)
│   ├── 1.1 산출물/작업 그룹 (Level 2)
│   │   ├── 1.1.1 세부 작업 (Level 3 = Work Package)
│   │   ├── 1.1.2 세부 작업
│   │   └── 1.1.3 세부 작업
│   └── 1.2 산출물/작업 그룹
│       ├── 1.2.1 세부 작업
│       └── 1.2.2 세부 작업
├── 2. 주요 단계
│   └── ...
└── 3. 주요 단계
    └── ...
```

## 3. 진행률 대시보드

### 3.1 전체 현황

```markdown
## 프로젝트 대시보드: [프로젝트명]
최종 업데이트: ____년 __월 __일

### 전체 진행률
████████████████░░░░ 80% (16/20 작업 완료)

### 단계별 현황
| 단계 | 진행률 | 상태 |
|------|--------|------|
| 기획 | ████████████████████ 100% | ✅ 완료 |
| 설계 | ████████████████████ 100% | ✅ 완료 |
| 개발 | ████████████░░░░░░░░ 60% | 🔄 진행중 |
| 테스트 | ░░░░░░░░░░░░░░░░░░░░ 0% | ⏳ 대기 |
| 오픈 | ░░░░░░░░░░░░░░░░░░░░ 0% | ⏳ 대기 |

### 핵심 지표
- 예정 완료일: ____.__.__
- 예상 완료일: ____.__.__  (지연: __일)
- 잔여 작업: __건
- 이슈: __건 (미해결: __건)
```

## 5. RACI 매트릭스 (역할 책임 배분)

```
R=Responsible(실행), A=Accountable(최종책임), C=Consulted(자문), I=Informed(통보)

| 작업 \ 담당 | PM | 기획팀 | 개발팀 | 디자인팀 | QA | 경영진 |
|-------------|-----|--------|--------|----------|-----|--------|
| 요구사항 정의 | A | R | C | C | I | I |
| 화면 설계 | A | C | C | R | I | I |
| 개발 | A | I | R | C | I | I |
| 테스트 | A | I | C | I | R | I |
| 최종 승인 | C | I | I | I | I | A |
```
